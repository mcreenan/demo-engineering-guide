This is the living engineering guide for Demo Co — a single, self-contained page that summarizes how we build, run, and reason about our systems. It is written for the people and agents working in this repo, and it is built from source: you change the content files and rebuild, never the generated HTML.

In shortEdit a section's HTML fragment, run pal build, and the guide reassembles itself. The guide never contradicts the source of truth — if a change conflicts with a source, fix the source first.

What this guide is

README.md (this project) AGENTS.md (this project)

It is an engineering handbook for Demo Co, rendered as one self-contained HTML page built with the palimpsest tool. It is meant to be read end-to-end when you are new and dipped into as a reference afterward.

The page ships with navigation and reading aids so you can find things fast:

  • A two-level table of contents and scrollspy that tracks where you are.
  • Section filtering to narrow the page to what you care about.
  • Pan/zoom Mermaid diagrams for architecture and flows.
  • Linked repo and code references that resolve to the real source.
  • An in-page Suggest a change flow that generates a ready-to-paste agent prompt.

Common commands

The everyday workflow is three commands:

pal build      # reassemble the guide
pal dev        # build, serve, and live-reload on change
pal validate   # lint section sources against the house style

Other commands exist for less frequent tasks:

  • pal outline and pal draft — scaffold structure and section drafts.
  • pal theme import — bring in design tokens.
  • pal publish — publish the built guide.
  • pal source add|list|remove — manage the registered sources of truth.
  • pal eject — take ownership of the engine's shell and behavior.

Making a change

Change a section's content
Edit its sections/<n>-<slug>.html fragment, then run pal build.
Add, move, or rename sections
Edit the structure block in palimpsest.config.json, then rebuild.
Adjust the look
Change the semantic tokens in theme/tokens.css.

The source-of-truth principle

AGENTS.md (this project) Engineering Wiki Help Center

This guide summarizes authoritative sources; it must never contradict them. State only what the sources support, and when unsure, hedge or omit. The registered sources are recorded in palimpsest.config.json and managed with pal source add|list|remove.

Engineering WikiADRs, runbooks, and process.

What the product looks like and how the UI behaves.

Each source is authoritative in its lane: the code (, , ) is authoritative for behavior, the Engineering Wiki for ADRs and process, and the Help Center for product-facing behavior. On a Help Center conflict, update the help article first, then the guide.

How to use it — and how to suggest changes

AGENTS.md (this project)

Read it top to bottom your first time through, then use the TOC, scrollspy, and filtering to jump back to a topic when you need it. The in-page Suggest a change feature turns an idea into a ready-to-paste agent prompt that follows the rules below.

Every suggestion is checked against the source of truth first:

  1. If it conflicts with a source. Stop — do not edit the guide. Record it in changelog/ as “blocked — conflicts with source” and say which source to update first. The guide is updated only after the source is.
  2. If it is consistent with sources. Log it in changelog/CHANGELOG.md with status “applied” and an ISO timestamp, edit the named sections/<n>-<slug>.html fragment, then run pal build.
Wording-only edits are exemptPure clarity, structure, or phrasing changes that alter no fact do not need a source check — just make them and rebuild.
