Run the whole stack locally with two commands: make dev in (migrates, seeds, then starts every backend service with hot reload) and pnpm dev in (runs the dashboard on port 3000 and the admin app on port 3001).

  • Prerequisites. Node >= 20, a local Postgres, and the DATABASE_URL env var. also needs pnpm (pinned pnpm@9.1.0).
  • Backend. make dev brings up all services against a seeded local Postgres with hot reload.
  • Frontend. pnpm install then pnpm dev runs every app in parallel; the apps expect the gateway at NEXT_PUBLIC_API_URL (defaults to http://localhost:3000).
  • Fresh laptop? The clone-all-three sequence lives in Engineering Onboarding.

CommandWhat it does
make devRuns make migrate and make seed, then starts every service with hot reload.
make migrateApplies each service's migrations in filename order (accounts, billing, usage, notifications). See Database Migrations for mechanics.
make seedLoads the fixture via psql "$DATABASE_URL" -f scripts/seed.sql.
make testRuns vitest across all workspaces. See Testing.
make lintRuns eslint over services.
make cleanRemoves built dist output and node_modules.

A typical first run looks like this:

export DATABASE_URL=postgres://localhost/platform
make dev

migrate.sh applies every .sql file in a directory in filename order with -v ON_ERROR_STOP=1, so a bad migration halts the run. You can target one service directly:

./scripts/migrate.sh services/accounts/migrations

The seed fixture

make seed loads a small, predictable dataset so you have something to develop against:

  • One org. "Acme Robotics" on the Team plan.
  • Three users. ada@acme.test (owner), grace@acme.test (admin), alan@acme.test (member).
  • One invoice. An open invoice for $49 (4900 cents) covering the current month.
Team plan is implied, not linkedThe seed file describes the org as being on the Team plan and inserts a matching $49 open invoice, but it does not write a plan row or an org-to-plan link. Treat the Team association as fixture intent rather than seeded data.

For service internals — the service classes, typecheck, and test mechanics — see Backend Development.

web (frontend)

web/package.json web/README.md

The frontend is a pnpm monorepo. Install once, then run every app in parallel:

pnpm install
pnpm dev
CommandWhat it does
pnpm devRuns every app in parallel — dashboard on :3000, admin on :3001.
pnpm --filter dashboard devRuns just one app.
pnpm testRuns vitest. See Testing.
pnpm lintRuns eslint over apps and packages.

The apps reach the backend only through the gateway, configured by two env vars:

NEXT_PUBLIC_API_URL
Gateway base URL. Defaults to http://localhost:3000 — the local gateway — so it usually needs no setting in development.
NEXT_PUBLIC_ENV
One of development, staging, or production.
Run both stacks togetherBecause NEXT_PUBLIC_API_URL defaults to the local gateway on port 3000, starting make dev in one terminal and pnpm dev in another gives you a fully wired local environment. For the api-client and the no-direct-fetch rules, see Frontend Development.
